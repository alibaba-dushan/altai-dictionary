# Алтайский словарь — веб-поиск

Статическое веб-приложение для поиска по словарям алтайского языка. Никакого
бэкенда: данные лежат в JSON, всё работает в браузере.

## Возможности

- **Живой поиск** — результаты обновляются прямо во время ввода, без кнопки.
- **Три режима** — по заглавному слову, по переводу/описанию или по всей статье.
- **Морфология русского запроса.** В режимах «по переводу» и «по всей статье»
  запрос сопоставляется по основе слова, поэтому `бросать` находит и `бросает`,
  `бросил`, `бросают`. Для алтайских заглавных слов морфология не применяется —
  там за варианты форм отвечают чередования (см. ниже).
- **Чередования звуков.** Запрос понимает, что гласные (и часть согласных)
  могут чередоваться: `гар` находит и `гор`. Группы:
  `а·е·о·ӧ`, `ы·и`, `у·ӱ`, `г·к`, `л·н·д·т`. Для согласных групп `г·к` и
  `л·н·д·т` первая буква запроса не расширяется (в начале слова они не чередуются).
- **Синонимы перевода.** Поиск по переводу можно расширить близкими по смыслу
  словами: запрос `кидать` найдёт и статьи с переводом `бросать`, `кинуть`,
  `швырять` и т. п. Найденные синонимы показываются чипами под строкой поиска —
  по ним можно кликнуть.
- **Подсветка совпадений.** В результатах подсвечивается сам запрос, его
  словоформы и найденные синонимы (тоже во всех словоформах).
- **Структура статьи.** Длинная статья разбирается на значения (`1)`, `2)`…),
  грамматические блоки (`1.`, `2.` — части речи / омонимы) и фразеологию
  (после знака `♦`); пометы и отсылки `см.`/`ср.` выделяются стилем.
- **Регулярные выражения** — `^аай`, `к[ао]л`, `.*чы$`.
- **Несколько словарей** — переключение между словарями и режим «все словари»,
  который ищет сразу по всем источникам.

## Запуск

Приложение полностью статическое. Из папки проекта:

```bash
python3 -m http.server 8000
```

и открыть `http://localhost:8000`. Подойдёт любой статический сервер
(`npx serve`, `php -S`, nginx и т. п.).

> Двойной клик по `index.html` обычно не срабатывает: браузеры запрещают
> `fetch()` локальных файлов из `file://`. Нужен именно HTTP-сервер.

Никаких зависимостей для запуска не требуется — только браузер.

## Состав репозитория

| Путь | Назначение |
|------|------------|
| `index.html` | разметка интерфейса |
| `styles.css` | оформление |
| `app.js` | вся логика: загрузка, поиск, морфология, синонимы, разбор статьи |
| `build_synonyms.py` | офлайн-сборка индекса синонимов (нужен только при пересборке) |
| `data/*.json` | словари, манифест и индекс синонимов |
| `README.md` | этот файл |

### Словари (папка `data/`)

| Файл | Словарь |
|------|---------|
| `altai_russian_dictionary_2018.json` | Алтайско-русский словарь, 2018 (основной) |
| `russian_altai_dictionary_2015_tom1_a_o.json` | Русско-алтайский, том I (А–О), 2015 |
| `russian_altai_dictionary_2016_tom2_p_ya.json` | Русско-алтайский, том II (П–Я), 2016 |
| `altai_ethnographic_dictionary_2023.json` | Алтайский этнографический словарь, 2023 |
| `manifest.json` | список словарей для интерфейса |
| `synonym_index.json` | индекс русских синонимов (см. ниже) |

Каждый словарь — это объект с полем `entries`. У двуязычных словарей запись
содержит `headword` / `headword_normalized`, `translation_text`, страницы и
перекрёстные ссылки; у этнографического — `term` / `term_normalized`,
`variants` и `entry_text`. Приложение учитывает обе схемы.

## Как устроены синонимы

`synonym_index.json` собирается офлайн скриптом `build_synonyms.py` из тезауруса
**RuWordNet 2021** (синсеты — группы настоящих синонимов, размеченные
лингвистами). Тезаурус выбран вместо эмбеддингов осознанно: ближайшие соседи в
векторных моделях подмешивают антонимы и ко-гипонимы (`бросать` → `ловить`),
а синсеты дают чистую синонимию.

Точность держится на нескольких приёмах:

1. Кандидаты берутся только из синсетов, где есть само слово запроса.
2. Каждый кандидат взвешивается по «тесноте» `sum(1 / размер_синсета)` — слово
   из маленького сфокусированного синсета ближе по смыслу, чем слово из
   большого размытого. Слабый хвост отсекается относительным порогом.
3. Остаются только слова, чья основа реально встречается в текстах словарей, —
   так в подсказках нет слов, которых в словаре всё равно не найти.
4. Для частотных слов (`говорить`, `красивый`, `бросать`…) есть небольшой
   выверенный вручную список-сид, который имеет приоритет; остальное добирается
   из тезауруса.

При поиске синонимы сопоставляются по основе слова, поэтому лемма `бросать`
находит и словоформы `бросал`, `бросив`, `бросаются`. Индекс загружается лениво —
только когда включён флажок «синонимы», — чтобы не замедлять открытие словаря.

### Пересборка индекса

Нужны база `ruwordnet-2021.db` (из проекта
[python-ruwordnet](https://github.com/avidale/python-ruwordnet)) и собранные
JSON-словари в `data/`. Скрипт использует только стандартную библиотеку Python 3:

```bash
python3 build_synonyms.py --db /путь/к/ruwordnet-2021.db
```

Путь к базе можно задать и переменной окружения `RUWORDNET_DB`. Результат
записывается в `data/synonym_index.json`.

## Источники данных

Словарные данные извлечены из опубликованных словарей алтайского языка
(А. Э. Чумакаев и др., 2015/2016/2018; этнографический словарь, 2023) и
предназначены для исследовательского и образовательного использования. Права на
исходные словари принадлежат их авторам и издателям.
